# Design Sprint Facilitator — Reference

## Sprint Phase Details

### DISCOVER — Activities

- Review existing BRD and knowledge base
- Identify business problems and market opportunities
- Define business goals and success metrics
- Clarify constraints: budget, timeline, PH market context
- Document assumptions and risks

### DESIGN — Activities

- Map core user journeys
- Break features into epics and stories
- Draft initial user stories with acceptance criteria
- Identify system boundaries and integration points
- Produce draft architecture overview

### DECIDE — Activities

- Prioritize MVP scope
- Refine user stories and acceptance criteria
- Identify technical stories and dependencies
- Decide backlog vs post-MVP items
- Finalize sprint-ready backlog

## PH Market Context Checklist

When applicable, consider:
- Philippine Peso (PHP) for all financials
- Local payment methods (GCash, PayMaya, bank transfer)
- Data residency / localization requirements
- Regulatory considerations (BSP, data privacy)
- Language (Filipino/English) for UX
- Mobile-first adoption patterns in PH
